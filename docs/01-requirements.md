# 需求分析文档

> 万能视频下载站（Universal Video Downloader）

## 1. 背景

主流视频平台经常存在以下下载痛点：

- 完全不支持下载（Twitter / X、Instagram、TikTok 等）
- 支持下载但有限制（清晰度、水印、必须 App、必须会员）
- 不支持批量下载或合集打包
- 手机端没有桌面客户端可用

需要一个一站式 Web 应用，PC + 移动端都能跑，输入 URL 一键拿到原画质视频，并附加 AI 增值能力（字幕翻译 / 视频总结）来支撑付费转化。

## 2. 目标用户

| 用户画像 | 核心诉求 |
|---|---|
| 创作者 / 剪辑师 | 批量素材采集、高清下载、字幕导出 |
| 知识工作者 | 视频总结、字幕翻译、随时随地看 |
| 普通学习者 | 手机端备份课程、追番、保留喜欢的内容 |

## 3. 功能范围（MVP = 阶段 3）

### 3.1 P0 核心（必须）

- F1 单视频解析：粘贴 URL → 拿到标题 / 缩略图 / 时长 / 作者 / 全部可用格式
- F2 格式选择下载：选清晰度 / 编码 / 仅音频，浏览器边下边推不占服务器盘
- F3 字幕导出：抓取平台原生字幕（含自动字幕），导出 SRT
- F4 批量解析：一次粘贴多条 URL，并发返回各自的可下载格式
- F5 移动端适配：响应式布局，手机粘贴 → 下载到相册可用

### 3.2 P1 付费增值（与 P0 同期上线，先做接口与按钮）

- F6 AI 视频总结：基于字幕 → LLM 生成中文要点 + 时间线
- F7 字幕翻译：把字幕翻译到指定语种，保留 SRT 时间戳

### 3.3 P2 预留（仅 UI 占位，本期不实现）

- F8 付费墙：定价页 + 升级 Pro 按钮（无真实支付）
- F9 用户系统 / 历史记录

### 3.4 明确不做

- 不做用户系统、数据库、真实支付
- 不集成 Whisper / 自有语音识别
- 不做合集打包 zip（前端逐条下载）
- 不做服务器存储下载产物（流式直推）

## 4. 非功能性需求

| 维度 | 要求 |
|---|---|
| 平台覆盖 | 借助 yt-dlp，覆盖 1000+ 主流站点（YouTube / Bilibili / TikTok / 抖音 / X / Instagram / Twitch / 微博 / 小红书 等） |
| 性能 | 解析 < 5s（90 分位）、下载边下边推无中间盘占用 |
| 体验 | 一次粘贴即用、无强制注册、无广告、移动端可用 |
| 部署 | 单机即可跑（FastAPI + Vite 静态），无数据库 |
| 安全 | 仅做转发，不留视频文件；LLM key 走环境变量不进前端 |

## 5. 关键风险

| 风险 | 应对 |
|---|---|
| 平台反爬 / 频控 | 直接跟随 yt-dlp 升级，不自研抓取逻辑 |
| 大文件下载占内存 | 用 `StreamingResponse` 边下边推 |
| 字幕缺失 | 缺失时前端友好提示，AI 总结/翻译需先拉到字幕 |
| LLM 成本 | 环境变量切 DeepSeek 等便宜模型，预留次数限制接口 |
| 版权 | 首页+底部声明：仅用于个人合法用途，不存储 |

## 6. 阶段划分

- 阶段 1（已完成）：需求分析、人工方案确认
- 阶段 2（已完成）：技术方案设计
- **阶段 3（本期）**：核心业务流程跑通 = F1~F7 全部上线，UI 一次到位
- 阶段 4（后续）：付费墙 + 用户系统 + 数据库
- 阶段 5（后续）：自有 Whisper 转录、合集 zip 打包、CDN 加速
